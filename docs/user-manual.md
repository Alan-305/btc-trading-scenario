# BTC Trading Scenario ユーザーマニュアル

初心者向けに、アプリの特徴・画面の見方・基本的な操作方法を説明します。

**本番URL:** https://nexus-btc-trading.web.app

---

## 1. このアプリでできること

BTC Trading Scenario は、**複数の市場データとあなたが登録した調査メモ**をもとに、

- **今日・今週・今月・半減期まで** の4つの時間軸でシナリオを表示
- **エントリー帯・利確・損切り** の目安を提示
- **価格の流れ（予測線）** をチャートで可視化
- **調査メモ（YouTube・記事など）** を整理し、シナリオ判断に反映
- **トレード日誌** で記録・振り返り

を行う **参考ツール** です。

> **重要:** 本アプリは投資助言ではありません。表示される数値やシナリオはすべて「参考情報」です。最終的な売買判断は、必ずご自身のルールとリスク許容度で行ってください。

---

## 2. ログイン

1. ブラウザで https://nexus-btc-trading.web.app を開く
2. **Google でログイン** を押す
3. ログイン後、ダッシュボードが表示されます

ログインすると、**シナリオの保存・調査メモ・トレード日誌** が使えるようになります（データはあなたのアカウントに紐づいて保存されます）。

---

## 3. シナリオ生成・価格予想に使うデータ

「再分析」ボタンを押すと、以下のデータを **まとめて** 取得し、方向・エントリー帯・予測線・解説文を計算します。

### 3.1 自動で取得する市場データ

| データ | 取得元（例） | シナリオへの使われ方 |
|--------|--------------|----------------------|
| **基準価格** | WhiteBIT の BTC/USDT | エントリー・利確・損切り・予測線の起点 |
| **取引所価格・板** | WhiteBIT / Binance / bitbank / Coinbase | 板の偏り（買い/売りどちらが厚いか）、スプレッド |
| **取引所間乖離** | 上記取引所の価格差 | 方向判断・解説文 |
| **恐怖・強欲指数** | Alternative.me | 上昇/下降のスコア |
| **デリバティブ** | Binance / Bybit / OKX / WhiteBIT 等 | ファンディングレート、L/S比率 |
| **テクニカル（4時間足）** | Binance ローソク足 | RSI、EMA、MACD、BB、サポ/レジ → 方向・エントリー調整 |
| **リスクゾーン** | Funding・L/S・板ヒートマップ | 清算帯・スクイズ帯の推定 → 損切り調整 |
| **板ヒートマップ** | 各取引所の板 | 支持/抵抗の厚い価格帯 → 方向・エントリー調整 |
| **マーケットセッション** | 日本/欧州/米国の時間帯 | 流動性・エントリー時間帯 → 信頼度・解説文 |

### 3.2 あなたが登録するデータ（シナリオ分析データ）

| 条件 | 反映されるか |
|------|--------------|
| ステータスが **有効（active）** | 必要 |
| **「今後のシナリオ分析に使う」** にチェック | 必要 |
| **要約（1行）** が入力済み | 必要 |
| 想定トレンド（上昇/下降/レンジ）を設定 | 方向判断に強く反映 |

YouTube の場合、**字幕は自動取得されません**。URL と動画タイトルに加え、**補足テキストに要点を貼る** と精度が上がります。

### 3.3 シナリオ生成に **使わない** データ

次のデータは **画面に表示・記録** されますが、**新しいシナリオの計算には直接は使いません**。

- トレード日誌のメモ・画像（振り返り用）
- 保存したシナリオスナップショット（過去予測の検証用）
- 日誌の参考リンク（ブックマーク用）

### 3.4 計算の仕組み（イメージ）

```
市場データ + 調査メモ
    ↓
ルールベースの方向・価格帯計算
    ↓
Gemini による「本日タブ」の日本語解説（API利用可能時）
    ↓
画面に表示
```

- **予測線（青い線）** は機械学習の長期予測ではなく、**現在のトレンド方向に沿った目安** です。
- **今週 / 今月 / 半減期** タブの解説は、同じ方向判断を **時間軸ごとに広げた目安** です（本日タブのみ Gemini 解説）。

シナリオカード下部の **「統合データ: …」** 行で、今回どのデータが使われたか確認できます。

## 4. 画面の見方（上から順）

ダッシュボードは **上ほど「判断の材料」、下ほど「記録・振り返り」** という構成です。

### 4.1 ヘッダー

| 要素 | 意味 |
|------|------|
| **Google でログイン / ログアウト** | アカウント切り替え |
| **日誌にも記録** | シナリオ保存時に日誌へもコピーするか |
| **シナリオを保存** | 今のシナリオを履歴に残す |
| **再分析** | 最新データでシナリオを再計算（約60秒ごとにも自動更新） |

「再分析」中は **「分析中…」** と表示されます。

---

### 4.2 シナリオ分析データ

YouTube・記事・テキストなど、**自分で集めた情報** を登録するエリアです。

- **分析に使う: N 件** … シナリオに反映される件数
- 一覧の **「編集」** … 登録後に AI 要約を生成し直せる
- **設定** … 新規追加時のデフォルト（分析に含めるか、古いデータの目安日数）

---

### 4.3 シナリオカード（最重要）

| 要素 | 読み方 |
|------|--------|
| **本日 / 今週 / 今月 / 半減期** タブ | 時間軸を切り替える。チャートの予測線も連動 |
| **上昇寄り / 下降寄り / レンジ** | 総合判断の方向 |
| **本文（日本語）** | エントリー・利確・損切りを含む解説 |
| **信頼度** | 0〜100%。高いほど各指標が同じ方向を示している |
| **統合データ** | テクニカル・リスクゾーン・調査メモなど、今回使ったデータ |

---

### 4.4 世界市場の時間帯

日本・欧州・米国の **株式市場の開閉** と、BTC の **流動性が高い時間帯** を示します。

- **エントリーのヒント** … 「今はエントリー検討に向く時間帯」など
- WhiteBIT は欧州〜米国時間、bitbank は日本の日中が参考になりやすい、という目安

---

### 4.5 ローソク足チャート

- **足** … 1時間 / 4時間 / 1日 などを切り替え
- **EMA・ボリンジャー** … テクニカル分析パネルと連動
- **サポート / レジスタンス** … 水平線
- **清算帯・スクイズ帯** … リスクゾーンパネルと連動

TradingView リンクから、同じ銘柄を詳しく見られます。

---

### 4.6 テクニカル分析 / リスクゾーン

**テクニカル分析:** RSI、EMA、MACD、ボリンジャーなどの要約（日本語）。

**リスクゾーン:** ロング清算帯・ショートスクイズ帯の **推定価格帯**（Funding・L/S 等から算出）。

---

### 4.7 センチメント系（4枚パネル）

| パネル | 内容 |
|--------|------|
| **恐怖・強欲指数** | 0（極度の恐怖）〜 100（極度の強欲） |
| **取引所乖離** | WhiteBIT 基準の各取引所との価格差 |
| **デリバティブ** | Funding、建玉、L/S など |
| **板ヒートマップ** | 価格帯ごとの買い/売りの厚さ |

---

### 4.8 エントリー判断と価格の流れ（チャート）

シナリオタブと **連動** したチャートです。

| 表示 | 意味 |
|------|------|
| **白/グレーの点線** | 過去の価格（参考） |
| **現在の価格** | 分析開始時点の WhiteBIT 基準価格 |
| **青い線** | 将来の目安（予測線）。タブごとに時間単位が変わる |
| **緑の帯** | エントリー帯（zone_low 〜 zone_high） |
| **オレンジ線** | 利確目安 |
| **赤線** | 損切り目安 |

ツールチップに **「過去の価格」「現在の価格」「将来の目安」** と表示されます。青い点がすべて「AI予測」という意味ではありません。

---

### 4.9 AI分析的中率

**過去に保存したシナリオ** が、その後の実際の値動きとどれだけ一致したかを集計します。7日間の Binance 足と照合します。

---

### 4.10 実トレード分析 / トレード日誌 / 保存履歴

- **実トレード分析:** 日誌に記録した **実際のトレード** と AI シナリオの比較
- **保存履歴:** 過去に「シナリオを保存」した一覧
- **トレード日誌:** メモ・タグ・画像・エントリー/決済価格・振り返りスコア

---

## 5. 基本的な操作手順

### 5.1 初めて使うとき（おすすめの流れ）

1. **Google でログイン**
2. 画面上部の **「再分析」** を押し、シナリオが表示されるまで待つ
3. **シナリオカード** で「本日」タブを読み、エントリー帯・利確・損切りを確認
4. 下の **価格チャート** で、緑の帯と青い予測線を照合
5. **マーケットセッション** で、今がエントリー向きの時間帯か確認
6. 必要なら **シナリオを保存**（日誌にも記録にチェック可）

---

### 5.2 調査メモ（YouTube 等）をシナリオに反映する

#### 新規登録

1. **シナリオ分析データ** → **「追加」**
2. **種類** … YouTube / URL / テキスト / PDF から選択
3. **タイトル** … わかりやすい名前
4. **URL** … YouTube や記事のリンク（種類が text 以外）
5. **補足テキスト** … 字幕・要点を貼る（YouTube は特に推奨）
6. **想定トレンド** … 上昇/下降/レンジ（任意だが設定推奨）
7. **「AI要約を生成」** … 要約（1行）が入るまで待つ  
   - 失敗時は赤文字で理由が表示されます（Gemini 利用上限など）
8. **「今後のシナリオ分析に使う」** にチェック
9. **「リストに追加」**

#### 登録だけして AI 要約を忘れた場合

1. 一覧の **「編集」**
2. 内容を確認 → **「AI要約を生成」**
3. **「更新する」**
4. **「再分析」**（調査メモを変えたあとは手動で押すと確実）

#### 反映されないときのチェックリスト

- [ ] 要約（1行）が空でないか
- [ ] 「今後のシナリオ分析に使う」にチェックがあるか
- [ ] ステータスが **有効** か（アーカイブされていないか）
- [ ] **再分析** を押したか

---

### 5.3 シナリオの期間を切り替える

1. シナリオカードの **本日 / 今週 / 今月 / 半減期** をクリック
2. 本文・エントリー帯・下の **価格チャート** が同じ期間に切り替わる
3. **半減期** は次回 BTC 半減期（2028年4月頃）までの長期目安

---

### 5.4 シナリオを保存する

1. ヘッダーの **「日誌にも記録」** をオン/オフ（好み）
2. **「シナリオを保存」**
3. 下部の **保存履歴** に追加される
4. **AI分析的中率** の計算対象になる（保存後の値動きと比較）

---

### 5.5 トレード日誌を書く

1. **トレード日誌** → **「新規」**
2. テンプレート（エントリー記録 / 決済記録 / 振り返りなど）を選択
3. 価格・メモ・タグ・画像を入力
4. **保存**

シナリオ保存時に「日誌にも記録」をオンにすると、シナリオ内容が日誌のたたき台としてコピーされます。

---

### 5.6 データを更新する

- **自動:** 約 **60秒** ごとに市場データとシナリオを更新
- **手動:** **「再分析」** を押す（調査メモを変更した直後はこちらが確実）
- **セッション時計:** 約30秒ごとに更新

## 6. 用語ミニ辞典

| 用語 | 意味 |
|------|------|
| **エントリー帯** | 新規ポジションを検討する価格の範囲 |
| **利確（TP）** | 利益確定の目安価格 |
| **損切り（SL）** | 損失を限定する目安価格 |
| **Funding** | 先物の資金調達率。正ならロング過多寄り、負ならショート過多寄り |
| **L/S比率** | ロングとショートの人数・建玉バランス |
| **恐怖・強欲指数** | 市場心理の参考指標（Alternative.me） |
| **半減期** | BTC のマイニング報酬が半分になるイベント（約4年周期） |

## 7. よくある質問

**Q. 予測線は必ず当たりますか？**  
A. いいえ。トレンド方向に沿った **参考の目安** です。急なニュースや清算で大きく動くことがあります。

**Q. YouTube を登録したのに要約が URL のコピーだけ**  
A. AI 要約が失敗した状態です。**編集 → AI要約を生成** を再度試してください。Gemini の利用上限の場合は時間をおくか、補足テキストに要点を貼ってください。

**Q. 「設定」と「編集」の違いは？**  
A. **設定** は全体の好み（新規のデフォルト等）。**編集** は1件ずつ内容や要約を直す画面です。

**Q. 調査メモの参考リンクはシナリオに効きますか？**  
A. **要約（1行）** と **想定トレンド** がシナリオに使われます。リンクだけでは反映されません。

**Q. 日誌はシナリオに影響しますか？**  
A. 現状、**新しいシナリオ生成には使いません**。振り返りと実トレード分析用です。

## 8. 注意事項

1. **投資助言ではありません。** 表示はすべて参考情報です。
2. **レバレッジ・全資金投入は慎重に。** 損切りは必ず自分で設定してください。
3. **API 障害・取引所メンテ** でデータが取得できない場合があります。
4. **Gemini（AI要約・解説）** は API キーと利用上限の状態により、テンプレート文にフォールバックすることがあります。
5. 重要な判断は **TradingView や取引所の公式チャート** でも必ず確認してください。
